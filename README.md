# Memory Reading

![Memory Reading — AI memory to your next book](.github/social-preview.svg)

**Your AI already knows what you are working through. Let that context choose your next book.**

Memory Reading recommends one timely book from what matters to the user now—not only from what they read before. The `find-next-book` skill combines the current request, relevant conversation context, available long-term memory, optional reading history, and verified book information.

> 不是继续“猜你喜欢”，而是根据你最近正在解决的问题，推荐此刻最值得读的一本书。

## Try it in 10 seconds

Install the Codex plugin marketplace:

```bash
codex plugin marketplace add Fov6363/memory-reading-skill
```

Then install **Memory Reading** from the plugin directory and start a new task:

```text
根据你对我的相关记忆，推荐我现在最值得读的一本书。
```

## A real recommendation

This example was generated by the skill from the public context of building and distributing this repository. No private memory or reading history was used.

> **Prompt**
>
> 我刚把一个 AI Skill 做出来，现在想让更多人知道。结合你对我当前阶段的了解，我下一本最该读什么？

**最推荐：《Traction》— Gabriel Weinberg、Justin Mares**

**为什么是现在：** 你已经越过了“把想法做出来”的阶段，当前瓶颈变成了找到可重复的分发方式。这本书针对的正是产品完成后如何获得用户，而不是继续打磨功能。

**它补什么：** 作者整理了 19 种获客渠道，并用 Bullseye 方法帮助团队筛选当下最值得测试的渠道，适合把“到处发链接”变成有优先级的小实验。

**建议读法：** 不必从头精读。先读 Bullseye 方法，再为这个 Skill 选 3 个候选渠道，各设计一个一周内能验证的实验。

**依据与不确定性：** 推荐只使用了当前对话中“已经做出可安装 Skill，正在解决分发”的信号，没有读取个人书架。书籍信息已通过 [Penguin Random House](https://www.penguinrandomhouse.com/books/319121/traction-by-gabriel-weinberg-and-justin-mares/) 核验。

## Principles

- Current user intent overrides remembered preferences.
- Only memory relevant to choosing a book may be used.
- Book titles, authors, premises, and availability must be verified.
- The workflow must degrade honestly when memory or reading integrations are unavailable.
- Persistent memory is not updated without explicit user intent or host confirmation.
